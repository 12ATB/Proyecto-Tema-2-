# Proyecto-Tema-2-
El cliente desea una aplicación web para gestionar los préstamos de libros del instituto. Quiere registrar libros y usuarios, controlar préstamos y devoluciones, y consultar el catálogo fácilmente. Busca una herramienta simple, rápida y segura para modernizar la biblioteca.
Metodología: SCRUM (Ágil)

**Motivos:**
Permite entregar versiones funcionales de la aplicación cada pocas semanas.
Se adapta a los cambios del cliente fácilmente.
Ideal para proyectos pequeños o medianos con equipos de 2-4 personas.
Favorece la comunicación continua entre cliente y equipo.

**Tamaño del grupo:**
Supongamos 3 miembros.

**Etapas principales:**

Planificación del Sprint

Se define qué historias de usuario se desarrollarán.

Desarrollo (Sprints de 2 semanas)

Se programan las funcionalidades definidas.

Daily Meeting

Reunión diaria de 15 minutos para revisar avances y obstáculos.

Revisión del Sprint

Se muestra al cliente el incremento de software desarrollado.

Retrospectiva del Sprint

Se analizan mejoras en la forma de trabajo.

Entrega final y documentación

Entrega del producto final con su documentación y pruebas.

**Roles:**

**Product Owner (PO):** Contacto con el cliente, define los requisitos. → Alumno 1

**Scrum Master (SM):** Asegura que se cumpla la metodología. → Alumno 2

**Development Team (Dev Team):** Diseña, programa y prueba el software. → Alumno 3 (y los demás)

**Reuniones y productos:**

**Reuniones:**

Daily (15 min)

Sprint Review (cada 2 semanas)

Retrospectiva (al final de cada Sprint)

**Productos:**

Product Backlog

Sprint Backlog

Incremento del producto (versión funcional)

Documentación técnica


**Requisitos funcionales:**

Registrar libros con título, autor, ISBN y estado.

Registrar usuarios (alumnos y profesores).

Gestionar préstamos y devoluciones.

Consultar disponibilidad de libros.

Mostrar historial de préstamos por usuario.

Panel para el bibliotecario con opciones administrativas.

Sistema de autenticación básico.

**Requisitos no funcionales:**

Interfaz web simple y responsive.

Acceso solo dentro de la red del instituto.

Base de datos segura y respaldada.

Tiempo de respuesta < 2 segundos por operación.

Cumplimiento de la LOPD (protección de datos).

Documentación completa y mantenible.

**Resumen de ejecución del proyecto**

**Semana 1-2:** Análisis de requisitos y diseño inicial (Product Backlog).

**Semana 3-4:** Sprint 1 → Implementación del registro de libros y usuarios.

**Semana 5-6:** Sprint 2 → Módulo de préstamos y devoluciones.

**Semana 7-8:** Sprint 3 → Interfaz final y panel de administración.

**Semana 9:** Pruebas completas, documentación y entrega final.

**Semana 10:** Presentación al cliente y cierre del proyecto.

**Resultado esperado**

Una aplicación web funcional que gestione los préstamos de la biblioteca del centro, fácil de usar, segura y documentada, desarrollada según los principios de Ingeniería del Software y metodología SCRUM.
